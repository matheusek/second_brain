# Roadmap

## Phase 1: Solid Workbench

Goal: turn the current prototype into a reliable daily-use tool.

### Outcomes

- stable conversations
- basic persistence
- useful file context
- clearer work modes
- better response UX

### Deliverables

- persistent chat history
- conversation list and switching
- file attachments for text and code
- real-time streaming
- retry and edit-last-message
- provider/model persistence per conversation
- better step execution states

## Phase 2: Context and Research

Goal: make the system better at understanding real work context.

### Outcomes

- conversations grounded in project material
- internet-assisted research when needed
- source-aware exploration and decision support

### Deliverables

- basic file ingestion
- image attachment support
- PDF support
- web search toggle
- source list in responses
- saved context bundles
- export to markdown

## Phase 3: Memory

Goal: make the system remember useful things over time.

### Outcomes

- durable facts
- durable preferences
- reusable context across sessions

### Deliverables

- persistent memory entries
- memory list UI
- edit/delete memory
- save memory from chat
- memory categories
- memory priority
- memory-aware prompts

## Phase 4: Projects and Workspaces

Goal: organize thinking by domain, project, or initiative.

### Outcomes

- project-specific context
- cleaner separation between topics
- better long-running usage

### Deliverables

- workspace/project model
- conversations by workspace
- memory by workspace
- context files by workspace
- default provider/mode by workspace

## Phase 5: Scheduled Intelligence

Goal: support recurring reminders, recurring analyses, and autonomous background work.

### Outcomes

- reminders
- daily/weekly/monthly reports
- recurring strategic and market analysis

### Deliverables

- scheduled tasks
- recurrence rules
- date-based reminders
- recurring runs
- task execution log
- output inbox
- provider/mode selection for tasks
- web-enabled tasks

## Phase 6: Operational Layer

Goal: move from assistant to work operator.

### Outcomes

- task pipelines
- project-aware execution
- stronger coding workflows

### Deliverables

- multi-step task templates
- project summaries
- issue generation
- plan/checklist generation
- coding review flows
- architecture review flows
- strategy templates

## Phase 7: Access and Deployment

Goal: make the system easy to use from anywhere with the right level of safety.

### Outcomes

- easy local access
- easy LAN access
- easy remote access
- controlled authentication

### Deliverables

- simple auth
- stable production mode
- remote access workflow
- terminal access workflow
- service health and logs
- backup/export path
