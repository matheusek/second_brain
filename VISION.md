# Vision

`second_brain` is an AI workbench for thinking, deciding, building, remembering, and running recurring intelligence tasks.

It should evolve from a chat UI into a practical system for:

- programming
- strategy
- research
- memory
- recurring analysis
- personal operations

## Core idea

One place where the user can:

- talk to multiple AI providers
- switch work modes depending on intent
- attach context and files
- store durable memory
- schedule recurring tasks
- review outputs later

## Product pillars

### 1. Workbench

A focused interface for live thinking and production.

- chat
- provider switching
- mode switching
- step-based execution
- context attachments

### 2. Memory

Persistent facts, preferences, and context that should survive across sessions.

- personal memory
- work memory
- preference memory
- project memory

### 3. Automation

The system should run useful tasks on its own.

- reminders
- recurring reports
- scheduled research
- recurring market or project analysis

### 4. Inbox

Outputs from recurring tasks and important generated artifacts should be reviewable later.

- scheduled runs
- reminders
- saved analyses
- exported outputs

## Guiding principles

- minimal UI
- useful by default
- explicit control over providers
- local-first where possible
- remote-capable where useful
- no decorative complexity
- strong support for programming and strategy

## Non-goals for now

- large multi-user collaboration
- complex permissions model
- enterprise deployment
- heavy RAG stack too early

## Near-term identity

In the near term, `second_brain` should feel like:

- a personal AI workbench
- a programming and strategy copilot
- a memory + routine system

## Long-term identity

In the long term, `second_brain` can become:

- a local/private alternative to hosted AI workspaces
- a personal AI operating layer
- a task runner for recurring thinking and decision workflows
